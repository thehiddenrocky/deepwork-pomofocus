# Contributing to PomoFocus: The Deep Work Edition

Thank you for your interest in contributing to PomoFocus! This project is built for performance engineers and focus-minded individuals who value their attention.

## Principles for Contribution

1.  **Keyboard-First:** Every feature must be accessible via keyboard shortcuts. Aim for 0% mouse dependency.
2.  **Privacy & Local-First:** No cloud sync, no tracking, no external telemetry. User data stays in their local CSV.
3.  **Zero-Latency UI:** Keep the UI lean. Avoid heavy frameworks or bloated dependencies. Prefer Vanilla JS and CSS.
4.  **Intentionality:** New features should help the user maintain focus, not distract them.

## Getting Started

1.  Fork the repository.
2.  Clone your fork: `git clone https://github.com/YOUR_USERNAME/PomoFocus.git`
3.  Install dependencies: `cd PomoFocus/app && npm install`
4.  Run in development mode: `npm run watch`

## Development Workflow

-   **Style:** Follow the existing Vanilla JS/CSS patterns.
-   **Debugging:** Remove all `console.log` statements before submitting a Pull Request.
-   **Documentation:** Update the `README.md` or `PHILOSOPHY.md` if your change introduces new concepts or shortcuts.

## Pull Request Process

1.  Create a new branch for your feature or fix.
2.  Ensure your code is clean and adheres to the principles above.
3.  Submit a PR with a clear description of the "Why" behind your change.

Let's build a distraction-free world, one focus session at a time.
